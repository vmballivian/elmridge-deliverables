1. DataBid Project Scraper

Live

scrape_databid.py

Automated extraction from DataBid's internal API
Logs into DataBid using Playwright (headless browser), extracts session cookies, then queries the internal REST API directly for project data. No fragile DOM scraping — this uses the same endpoints the DataBid dashboard uses internally.
API Endpoints
getDueProjects (87 projects)
getRecentProjects (43 projects)
Regions
Eastern Ontario + Southern Ontario
Schedule
Weekly (Sunday night)
Output
Raw Scraping Results tab in Google Sheets

Filters Applied

Patrick's criteria: "anything multi-unit, multi-level, multiplex type stuff" — buildings where developers and GCs need carpentry subcontractors for baseboard, casing, doors, and framing.

✓
Project Class: General Building only
✕
Excluded classes: Heavy/Civil (roads, sewers, utilities), Professional Consulting, Supply/Maintenance, Individual Trades
✓
Industries kept:
Residential Commercial Educational Medical
✕
Industries excluded:
Heavy/Highway/Utilities Power Generation Transportation Outdoor Light/Power
✕
Keyword exclusions (removed even if General Building):
pump station sewage ventilation hvac heating pump rooftop unit roof replacement window program fenestration solar / generator elevator fire alarm transit stop paving / asphalt
✓
Boost keywords (always kept regardless of industry):
residential condo apartment townhouse multi-unit mixed-use new school renovation community centre housing hospital hotel office framing
First run pulled 130 raw projects from the API. After filtering: 20 relevant leads written to the sheet. The other 110 were infrastructure, mechanical work, or consulting — not carpentry opportunities.

2. Ottawa Permits Scraper

Live

scrape_permits.py

City of Ottawa monthly construction permit data (open.ottawa.ca)
Downloads the monthly Excel file from Ottawa's ArcGIS open data portal automatically. Processes 7,489 permit records from 2024-2025, filters for multi-unit residential (4+ DU) and commercial construction, groups by contractor (one lead per company), and writes to the Google Sheet.
Data Source
open.ottawa.ca via ArcGIS API
Auto-download with --fetch flag
Raw Records
7,489 permits (2024-2025 dataset)
Schedule
Monthly (1st of month)
Output
Raw Scraping Results tab
All HIGH priority (Ottawa = drive-to)

Filters Applied

Patrick highlighted permits as especially valuable for the framing expansion (8-40 unit builds): "if we can see where permits are being approved, we can physically go to that site"

✓
Multi-unit residential: 4+ dwelling units (rowhouses, stacked rowhouses, apartments)
✓
Commercial/Institutional: tenant fit-ups, new construction, renovations with interior work
✓
Grouped by contractor: one row per company, largest project selected
✕
Excluded permit types:
Demolitions Pools Single-family (DU < 4) Semi-detached (DU 2-3)
✕
Excluded descriptions:
plumbing only electrical only mechanical only fire alarm sprinkler elevator hvac roofing solar / generator parking / paving backflow prevention
✕
Excluded contractor types (trade subs, not developers/GCs):
plumbing electrical hvac / heating mechanical glass / glazing roofing painting landscaping
✓
All Ottawa permits = HIGH priority. Patrick's team (Jerry) physically drives to these sites.
First run: 7,489 raw permits → 138 unique contractors after filtering → 131 new leads written to sheet (7 already existed from DataBid). Top contractors: Richcraft Homes, Falsetto Homes, Claridge Homes, Taggart, Mattamy.

3. Development Applications Scraper

Live

scrape_devapps.py

Ottawa development applications with applicant/owner identification (devapps.ottawa.ca)
This is the answer to Patrick's key question from Call 4: "We got to find a way to find out who the owner is... who's getting these permits approved." The City of Ottawa publishes development applications (zoning amendments, site plans, subdivisions) through a public REST API. Each application has an Application Summary PDF that names the applicant and property owner.
Data Source
devapps.ottawa.ca REST API
Public, no login required
Total Applications
919 active development applications
763 relevant types (site plan, zoning, subdivision)
Owner Extraction
Application Summary PDFs
~50% have PDFs with applicant/owner names
Schedule
Monthly (new applications appear regularly)

How It Works

1.
Fetches all dev applications from the REST API (919 records)
2.
Filters to Site Plan Control, Zoning Amendments, Subdivisions, Condominiums
3.
Fetches detail for each application + downloads Application Summary PDF
4.
Extracts Applicant and Owner names from PDF (e.g., "Keeper Co. c/o Bridgette Alchawa", "1850591 Ontario Ltd.")
5.
Includes city planner contact + project description in notes for outreach context
First run: 763 applications processed → 626 PDFs parsed → 241 new leads with identified owners/developers written to the sheet. These are planning-stage projects — the earliest signal available. Combined with permits (construction-stage) and DataBid (pre-construction), Patrick's team can see projects at every stage of the pipeline.

4. Contact Enrichment

Live

enrich_leads.py

Apollo API — find decision-makers at each company
Two-step strategy to minimize credit usage. Step 1 (free): search Apollo for people at the company matching target titles. Step 2 (1 credit): enrich the best contact to get email and phone. Only the top contact per company is enriched.
Target Titles
Project Manager, Estimator, VP Construction, Procurement Manager, Owner
Credit Usage
~1 credit per company
~416 credits/month on Basic
Schedule
Daily on weekdays
Cost
$49/month (Apollo Basic)

How It Works

1.
Reads Raw Scraping Results for rows with Status="New" and missing contact info
2.
Free search: Apollo People Search returns names, titles, LinkedIn URLs (no credits)
3.
Picks best contact by title priority (PM > Estimator > VP > Owner)
4.
Paid enrichment: Gets email + phone for top contact only (1 credit)
5.
Updates the sheet row with Contact Name, Title, Email, Phone, LinkedIn
First run: enriched 313 leads (84 new scraped + 229 HubSpot legacy warm/hot). 292 Apollo credits used. Hit rate: 69% for established companies, 22% for smaller local builders. Runs automatically Mon-Fri at 6am.

6. LinkedIn Sales Navigator

Phase 3

scrape_linkedin.py

Optional contact enrichment layer (supplements Apollo)
Processes CSV exports from LinkedIn Sales Navigator. Does not scrape LinkedIn directly to avoid account ban risk. The team exports leads from Sales Navigator, drops the CSV into the data/linkedin/ folder, and this script processes it into the Raw Scraping Results tab.

How it fits the pipeline

Sales Navigator is not for finding projects — DataBid and Ottawa Permits already handle that. It's a second way to find decision-makers at companies we already know about, supplementing Apollo.

Apollo (Primary enrichment)
Automated via API
Runs daily, no manual work
Returns email + phone
1 credit per contact (~416/mo)
Sales Navigator (Optional add-on)
Manual CSV export
Higher quality — see activity, mutual connections
Better for warm outreach style
Can spot development announcements in posts

Patrick's outreach style is referencing a specific project: "Hey, I've come across your project at so-and-so street... Just CC us when you send out your tender package." Sales Navigator lets you see who at Richcraft or Claridge is active on LinkedIn and tailor the approach — but Apollo already gets you the contact info to make that call.

Recommended: start with Apollo enrichment alone (included in Phase 2). Add Sales Navigator as a Phase 3 upgrade once the core pipeline is generating outreach-ready leads. Script is already built and ready.
